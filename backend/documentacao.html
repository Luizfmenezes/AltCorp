Visão Geral da API

Esta API fornece endpoints para gerenciar autenticação, inserção de dados (notas fiscais, ordens de serviço, itens, grupos e subgrupos de itens), alteração de dados e listagem de dados (estoque e itens). Todas as requisições devem ser feitas com o método especificado e o corpo da requisição deve ser um JSON válido.

A URL base para todos os endpoints é http://localhost:8000. Lembre-se de substituir pela URL real do seu servidor de API.

Autenticação

Todos os endpoints que interagem com o banco de dados (exceto o próprio endpoint de autenticação) esperam que as credenciais do usuário (`sistema`, `usuario`, `senha`) sejam fornecidas no corpo da requisição JSON para validação.

1. Autenticação

1.1. POST /autenticar

Autentica um usuário fornecendo suas credenciais de sistema, usuário e senha.

Parâmetros da Requisição (Payload JSON):

Campo Tipo Obrigatório Descrição
sistema integer Sim Identificador do sistema.
usuario string Sim Nome de usuário para autenticação.
senha string Sim Senha do usuário.

Exemplo de Requisição (JavaScript - Fetch API):


async function autenticarUsuario() {
    // Substitua 'http://localhost:8000' pela URL base da sua API
    const url = 'http://localhost:8000/autenticar';
    const data = {
        "sistema": 1,
        "usuario": "exemplo.usuario",
        "senha": "senha123"
    };

    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(data)
        });

        const result = await response.json();
        if (response.ok) {
            console.log("Autenticação bem-sucedida:", result);
            // Prossiga com a lógica do seu aplicativo, como armazenar informações de sessão
        } else {
            console.error("Erro na autenticação:", result.mensagem || result.detail);
        }
        return result;
    } catch (error) {
        console.error("Erro ao conectar à API:", error);
        throw error;
    }
}

2. Inserção de Dados

2.1. POST /inserir/notafiscal

Insere uma nova nota fiscal no sistema, incluindo seus itens associados.

Parâmetros da Requisição (Payload JSON):

Campo Tipo Obrigatório Descrição
sistemaintegerSimIdentificador do sistema.
usuariostringSimNome de usuário para autenticação.
senhastringSimSenha do usuário.
statusintegerSimStatus da nota fiscal.
nfintegerSimNúmero da nota fiscal.
valorbrutofloatSimValor bruto da nota fiscal.
datacomprastringSimData da compra no formato ISO 8601 (YYYY-MM-DDTHH:MM:SS).
obsstringOpcionalObservações adicionais.
itensarray<object>SimLista de itens da nota fiscal. Cada objeto de item deve conter:
  • item (integer): ID do item.
  • status (integer): Status do item na nota fiscal.
  • qtd (integer): Quantidade do item.
  • tipofaturamento (integer): Tipo de faturamento (1 para VALORUNITARIO, outro para VALORCONJUNTO).
  • valorfaturamento (float): Valor de faturamento do item.
  • obs (string): Observação do item.

Exemplo de Requisição (JavaScript - Fetch API):


async function inserirNotaFiscal() {
    const url = 'http://localhost:8000/inserir/notafiscal';
    const data = {
        "sistema": 1,

        "obs": "Compra de materiais de escritório",
        "itens": [
            {
                "item": 101,
                "status": 1,
                "qtd": 5,
                "tipofaturamento": 1,
                "valorfaturamento": 10.50,
                "obs": "Canetas azuis"
            },
            {
                "item": 102,
                "status": 1,
                "qtd": 2,
                "tipofaturamento": 2,
                "valorfaturamento": 50.00,
                "obs": "Pacote de papel A4"
            }
        ]
    };

    try {
        const response = await fetch(url, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(data)
        });

        const result = await response.json();
        if (response.ok) {
            console.log("Nota fiscal inserida com sucesso:", result);
        } else {
            console.error("Erro ao inserir nota fiscal:", result.mensagem || result.detail);
        }
        return result;
    } catch (error) {
        console.error("Erro ao conectar à API:", error);
        throw error;
    }
}

2.2. POST /inserir/ordemservico_compra

Insere uma nova ordem de serviço de compra no sistema.

Parâmetros da Requisição (Payload JSON):

Campo Tipo Obrigatório Descrição
sistemaintegerSimIdentificador do sistema.
usuariostringSimNome de usuário para autenticação.
senhastringSimSenha do usuário.
statusintegerSimStatus da ordem de serviço (sempre 1 = ATIVO).
tipointegerSimTipo da ordem de serviço (sempre 1 = VENDA DIRETA).
obsstringOpcionalObservações adicionais (se não informado, será string vazia `""`).
solicitacaoservicoobjectSimObjeto da solicitação de serviço associada. Deve conter:
  • status (integer): Status da solicitação de serviço (sempre 2 = VENDIDO).
  • tipo (integer): Tipo da solicitação de serviço (sempre 3 = VENDA DIRETA).
  • cliente (integer ou boolean): ID do cliente ou `false` para cadastrar um novo cliente.
  • cliente_info (object): Informações do cliente, se `cliente` for `false`.
    • tipo (integer): Tipo de documento (1 para CPF, 2 para CNPJ).
    • nome (string): Nome do cliente.
    • cpfcnpj (string): CPF ou CNPJ do cliente (string vazia `""` para não cadastrar).
    • cep (string): CEP do cliente (string vazia `""` para não cadastrar).
    • telefone (string): Telefone do cliente (string vazia `""` para não cadastrar).
    • email (string): Email do cliente (string vazia `""` para não cadastrar).
  • descricao (string): Descrição da venda (sempre "Venda de produto").
  • margemlucro (float): Margem de lucro (ex: 20.6% de margem de lucro).
  • itens (array<object>): Lista de itens da solicitação (obrigatório).
    • item (integer): ID do item cadastrado.
    • quantidade (integer): Quantidade do item.
    • tipo (integer): Tipo (sempre 1 = venda).

Exemplo de Requisição (JavaScript - Fetch API):

                {

    try {

3. Alteração de Dados

3.1. PUT /alterar/item

Altera as informações de um item existente no sistema. Apenas os campos fornecidos serão atualizados.

Parâmetros da Requisição (Payload JSON):

Campo Tipo Obrigatório Descrição
sistemaintegerSimIdentificador do sistema.
usuariostringSimNome de usuário para autenticação.
senhastringSimSenha do usuário.
idintegerSimID do item a ser alterado.
descricaostringOpcionalNova descrição do item.
grupointeger ou booleanOpcionalNovo ID do grupo, ou `false` para não alterar.
subgrupointeger ou booleanOpcionalNovo ID do subgrupo, ou `false` para não alterar.
tipovalidadeinteger ou booleanOpcionalNovo tipo de validade, ou `false` para não alterar.
validadestring, integer ou booleanOpcionalNova validade, ou `false` para não alterar.
temgarantiainteger ou booleanOpcionalNovo indicador de garantia, ou `false` para não alterar.
garantiastring, integer ou booleanOpcionalNova duração da garantia, ou `false` para não alterar.

Exemplo de Requisição (JavaScript - Fetch API):


async function alterarItem() {
    const url = 'http://localhost:8000/alterar/item';
    const data = {
        "sistema": 1,
        "usuario": "exemplo.usuario",
        "senha": "senha123",
        "id": 103, // ID do item a ser alterado
        "descricao": "Processador Intel Core i7 14ª Gen",
        "grupo": false, // Não altera o grupo
        "subgrupo": 201, // Altera para o subgrupo com ID 201
        "tipovalidade": 0, // Sem validade
        "validade": "", // Vazio para sem validade
        "temgarantia": 1, // Com garantia
        "garantia": 36 // 36 meses de garantia
    };

    try {
        const response = await fetch(url, {
            method: 'PUT',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(data)
        });

        const result = await response.json();
        if (response.ok) {
            console.log("Item alterado com sucesso:", result);
        } else {
            console.error("Erro ao alterar item:", result.mensagem || result.detail);
        }
        return result;
    } catch (error) {
        console.error("Erro ao conectar à API:", error);
        throw error;
    }
}

Códigos de Status HTTP:

  • 200 OK: Requisição bem-sucedida.
  • 400 Bad Request: Campos obrigatórios ausentes ou inválidos.
  • 500 Internal Server Error: Erro interno no servidor ou no banco de dados.

5. Tratamento de Erros

A API retorna códigos de status HTTP para indicar o resultado da requisição e um corpo de resposta JSON com uma mensagem (`mensagem`) e, em alguns casos, um status (`status`) ou detalhes (`detail`) adicionais.

  • 200 OK: Indica que a requisição foi processada com sucesso. A resposta conterá os dados esperados ou uma mensagem de sucesso.
  • 400 Bad Request: Indica que a requisição não pôde ser entendida pelo servidor devido a sintaxe malformada ou parâmetros inválidos/ausentes. Verifique o payload JSON enviado.
  • 401 Unauthorized: (Potencialmente) Indica que a autenticação falhou ou não foi fornecida. Embora seus endpoints atuais retornem 200 com status 0 para falha de autenticação, é uma boa prática considerar 401 para o futuro.
  • 500 Internal Server Error: Indica um erro inesperado no servidor. Isso pode ocorrer devido a problemas no banco de dados, lógica interna da API ou outros problemas de infraestrutura.

Sempre verifique a propriedade response.ok no JavaScript após uma requisição `fetch` para determinar se a requisição HTTP foi bem-sucedida (status 2xx). Em seguida, inspecione o corpo da resposta JSON para mensagens específicas de erro da API.
